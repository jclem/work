work

A CLI for running parallel, isolated tasks against your projects.

work manages projects, environments, and tasks. Register a project, and work can spin up isolated environments (git worktrees by default) and run tasks in them via configurable providers. A background daemon handles preparation and execution. A built-in TUI lets you monitor everything.

How it works

The core workflow is:

  1. Register a project — point work at a git repository.
  2. Create a task — describe what needs to be done.
  3. work handles the rest — it creates an isolated environment, runs your configured task provider (e.g. Claude Code, Codex, a shell script), and captures all output.

You can run many tasks in parallel, each in its own isolated environment. Monitor them all from the TUI or stream logs from the CLI.

Install

Homebrew

brew install jclem/tap/work

From source

cargo install --git https://github.com/jclem/work

Quick start

# Start the daemon
work daemon start

# Register a project (defaults to cwd)
work project new

# Run a task
work task new "fix the login bug" --attach

This creates a git worktree, runs your configured task provider in it, and streams the output to your terminal. See Getting started for a full walkthrough.
