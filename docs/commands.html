Commands

Global flags

FlagEnv varDescription
--debug WORK_DEBUG Enable debug logging
--work-home PATH WORK_HOME Override all data/config/runtime paths

Output formats

Commands that list data accept --format with one of:

FormatDescription
humanAligned columns (default)
plainTab-separated, no headers
jsonJSON array or object

work project

work project new [name] [--path PATH]

Register a project. Defaults to the current directory name and path. 

# Register current directory
work project new

# Specify name and path
work project new my-api --path ~/src/api

work project list [--format FORMAT]

List all registered projects. Alias: work project ls.

work project remove NAME

Unregister a project. Alias: work project rm. Does not delete any files.

work task

work task new DESCRIPTION [options]

Create a new task. This prepares an environment and runs the configured task provider command.

OptionDescription
--provider NAME Task provider (falls back to default-task-provider in config)
--env-provider NAME Environment provider (falls back to default-environment-provider)
--project NAME Project name (defaults to project matching cwd)
--attach, -a Follow task logs after creation
--format FORMAT Output format 
# Use defaults
work task new "fix the login bug"

# Specify providers explicitly
work task new "add tests" --provider claude --env-provider sandbox

# Follow output
work task new "refactor auth module" --attach

work task list [--format FORMAT]

List all tasks. Alias: work task ls.

work task remove ID

Remove a task and its associated environment. Alias: work task rm.

work logs

work logs ID [--follow]

View output from a task.

# View completed task output
work logs 4Abc123xYz

# Stream in real time
work logs 4Abc123xYz --follow
work logs 4Abc123xYz -f

work env

Alias: work environment.

work env create [project] [--provider NAME]

Create a new environment and immediately claim it. The project defaults to the project matching your current directory.

work env prepare [project] [--provider NAME]

Create a new environment and add it to the pool. Pooled environments are not yet in use and can be claimed later by a task or manually.

work env claim [ID] [--provider NAME --project NAME]

Claim a pooled environment. Either specify an environment ID directly, or use --provider and --project to claim the next available match.

work env update ID

Update a pooled environment. For git-worktree, this fetches from origin and merges.

work env remove ID

Remove an environment. Alias: work env rm. Cleans up the worktree or calls the provider's remove action.

work env list [--format FORMAT]

List all environments. Alias: work env ls.

work env provider list

List all available environment providers (built-in and configured). Alias: work env provider ls.

work tui

Open the terminal UI. Shows tasks, projects, and environments with real-time updates via event streaming.

Keybindings:

KeyAction
TabSwitch between tabs
j / kNavigate up/down
g / GJump to top/bottom
d / uHalf-page down/up
EnterView task logs
DDelete selected item
qQuit

work config

work config edit

Open the config file in $EDITOR.

work daemon

work daemon start [--force]

Start the daemon process. Use --force to remove stale runtime files before starting.

work daemon install

Install the daemon as a macOS launchd LaunchAgent. It will start automatically on login.

work daemon uninstall

Uninstall the LaunchAgent.

work completions

work completions SHELL

Generate shell completions. Supported shells: bash, zsh, fish, elvish, powershell.

work version

Print the version.
