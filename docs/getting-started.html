Getting started

1. Start the daemon

work uses a background daemon to manage environments and execute tasks. Start it directly: 

work daemon start

Or install it as a launchd service so it starts automatically on login (macOS): 

work daemon install

2. Register a project

Navigate to a git repository and register it: 

cd ~/src/my-project
work project new

This registers the current directory as a project named after the directory. You can also specify both: 

work project new my-project --path ~/src/my-project

Verify your projects:

$ work project list
NAME        PATH
my-project  /Users/you/src/my-project

3. Configure a task provider

Before creating tasks, configure a task provider. This tells work what command to run. Open the config: 

work config edit

Add a provider. Here's one that runs Claude Code: 

default-task-provider = "claude"
default-environment-provider = "git-worktree"

[tasks.providers.claude]
type = "command"
command = "claude"
args = ["-p", "{task_description}"]

The {task_description} placeholder is replaced with the task description you provide when creating a task.

4. Create a task

From within your project directory: 

work task new "fix the login bug"

This will:

  1. Create a git worktree for isolation
  2. Run claude -p "fix the login bug" inside the worktree
  3. Capture all output to a log file

Use --attach to stream logs to your terminal: 

work task new "add form validation to signup page" --attach

Or follow logs for an existing task: 

work logs <task-id> --follow

5. Monitor with the TUI

work tui

The terminal UI shows all tasks, projects, and environments with real-time updates. Navigate with tab, arrow keys, and vim-style bindings (j/k, g/G).

6. Run tasks in parallel

Each task gets its own environment, so you can run as many as you want concurrently: 

work task new "fix the login bug"
work task new "add unit tests for auth module"
work task new "update API documentation"

All three tasks run simultaneously in separate git worktrees. Monitor them in the TUI or check on any one with work logs.

7. Environment pooling

Creating environments takes time (cloning worktrees, installing dependencies). You can pre-warm a pool of environments so tasks start instantly: 

# Prepare some environments ahead of time
work env prepare
work env prepare
work env prepare

When you create a task, work will claim an available pooled environment instead of creating a new one. If no pooled environment is available, one is created on the fly.

Cleanup

Remove a task (and its environment): 

work task rm <task-id>

Remove a standalone environment: 

work env rm <env-id>

Unregister a project: 

work project rm my-project
